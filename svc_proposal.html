01 — Current State

Where Things Stand Today

Command Center 3.0 runs on AppSheet with Google Sheets as its backend. It handles core field operations — but the platform has reached its ceiling for what SVC needs next.

What works
Clock IN / Clock OUT for field workers. Active job tracking with PM and Supervisor assignments. Daily report submission. Report frequency settings per job (Daily, Hourly, Nights, Same Day).
Platform limitations
Generic UI with no design control. No mobile-first experience. Manual processes that should be automated. Hard to scale beyond current feature set. Vendor dependency on AppSheet.
Engagement gap
Field workers use the app because they have to — not because it's fast or satisfying. Low adoption leads to missing reports, delayed data, and manual follow-up from managers.
Data foundation
Core tables are solid — Contacts, Jobs, Companies, Users, Timesheets. This data migrates cleanly to the new platform. No rebuilding from scratch.

04 — Technology

Stack

Modern, proven tools — no proprietary platforms, no vendor lock-in. Built to own and scale independently.

⚡
Next.js
Frontend framework. Fast, mobile-first, installable on any phone as a PWA without an app store.
🔥
Firebase
Authentication, real-time database, and photo storage. User login, live data sync, and file uploads.
▲
Vercel
Deployment and hosting. Instant updates, global CDN, zero server management. Accessible by link on any device.
📊
Sheets API
Data migration from existing Google Sheets. Optional continued report export to spreadsheets.

06 — Future Scope

Possible Enhancements

Not in migration scope. These are features to evaluate after launch — each one adds a specific layer of engagement or operational visibility. To be scoped and prioritized with the team.

01
🔥
Report Streak
Workers build a daily reporting streak visible on their home screen. Consistent reporting becomes a habit, not a chore. Managers see who's consistent without asking.
Drives daily engagement without management follow-up
02
📍
Photo + GPS Verification
Clock IN requires a jobsite photo and confirms GPS coordinates match the job location. Prevents off-site clock-ins. Photo stored per entry for review.
Eliminates timesheet fraud and improves field accountability
03
📊
Job Score
Each job gets a live score based on reporting consistency, clock-in timing, and photo uploads. Admin sees which jobs are running well and which need attention — at a glance.
Replaces manual check-ins with automatic health indicators
04
⚡
Quick Problem Check-In
At Clock OUT, one question: "Any issues today?" — Yes or No. If yes, a short field opens for details and an optional photo. Field problems captured before they escalate.
Captures issues in real time, not after the fact
05
🎙
Voice Report
Workers speak their daily report instead of typing. App transcribes automatically. Designed for field conditions — gloves, tools, no time to type a paragraph.
Removes the biggest friction point in daily reporting
06
📡
Live Activity Feed
A real-time timeline for admin — who clocked in, which reports were submitted, which jobs have no activity today. Full operational pulse in one screen.
Full visibility without calling or chasing anyone
Built to last —
and to grow.
